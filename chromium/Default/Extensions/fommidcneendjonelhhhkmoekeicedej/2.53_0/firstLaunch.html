Thanks for using Hacker Vision!

Read this quick guide to learn how to use Hacker Vision and toggle it on and off for different websites.



Let's get started:

Visit your favorite website and click the h icon near the top of your screen to open the main menu.



The popup will launch and you will see the slider at the top that toggles HV on/off for all websites.

Below that you will see the URL of the current website you are looking at. Simply choose either "Dark" or "Normal" and HV will remember your settings for that site forever.

To set HV to be Dark or Normal by default, open HV on a new tab page and it will prompt you to "Set Global Default".
There's also a handy slider to temporarily Disable HV on all sites.

It's super useful if you're outside or in a bright room (think of it as a Daytime Mode).

I work hard keeping Hacker Vision up to date and if you love using it, please remember to spread the word using the share buttons.

That's all there is! If you have any questions, please visit the Frequently Asked Questions section.

Why doesn't HV work on this page or the Chrome Web Store, or New Tab?
For security purposes Google prevents all extensions from running on any url that begins with "chrome://" (including this page).

If you want to test this, the New Tab page actually has a url of "chrome://newtab", and if you type that in the URL bar, it will open a new tab.

If you want proof that it's working, just click here and the h icon in your browser bar to make sure it's turned on.

Sometimes I see images that look inverted, why is this?
This happens when a website sets a picture as a background-image instead of an image. Much like how the wallpaper on your desktop is a background image.

Because these are usually reserved as the background for the entire page, HV does its job and inverts it.

However sometimes sites will do this for thumbnails, and while it may not look pretty, it is not possible for HV to be perfect for 100% of pages on the internet. A human touch is still needed.

There is a brand new BETA feature in the works to fix these inverted images so stay tuned!
Can you detect if the website is bright and only apply HV to those pages?
Yes, I have tried this, and it works. But there are performance issues.

The issue is that HV cannot process the page to determine if it is bright or dark until after it has finished loading. Which means there is a very noticable lag time and a much longer white flash before the dark page displays.

I am currently working on refining this algorithm. If you have any suggestions, please email below.
Why is there sometimes a white flash before a page loads?
This is a known Google Chrome bug. It is caused because Google decided all websites would be "painted" on a white canvas. There is no way to bypass this.

However I have done a few tricks to help minimize it. However you may still experience it more on older or slower computers.

If you hate this as much as I do, please email or tweet at the Google Chrome team and tell them to change the default canvas to BLACK or DARK GREY.
Sometimes a video on Youtube is inverted full-screen but not vice-versa(or vice-versa)
Youtube is constantly changing their code and it is completely dependant upon your machine, operating system, and current login status.

I am constantly fixing this but hopefully it will settle down soon.

That being said, I am on Mac OSX and I never encounter this. If you do notice it, please email me a link to the page, and a copy of the page source code (right-click, 'View page source').
